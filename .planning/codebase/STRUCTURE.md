## Directory Purposes

**`skills/`:**
- Purpose: Agent Skills catalog — each subdirectory is an independently installable skill
- Contains: Skill definitions (SKILL.md), guidelines (AGENTS.md), and reference documentation
- Key files: `SKILL.md` (agent-facing specification), `AGENTS.md` (agent developer guidelines)
- Publish target: Skills are distributed via `npx skills add` to `~/.claude/skills/`, `~/.cursor/skills/`, etc.

**`skills/qodo-get-rules/`:**
- Purpose: Fetches all coding rules from Qodo API with pagination
- Contains: Skill definition, reference docs for pagination, output formatting, and repository scope detection
- Entry point: `SKILL.md`
- Reference files: `references/pagination.md`, `references/output-format.md`, `references/repository-scope.md`, `references/attribution.md`

**`skills/qodo-get-relevant-rules/`:**
- Purpose: Searches for rules most relevant to a specific coding task using semantic search
- Contains: Skill definition, reference docs for query generation and search endpoint contract
- Entry point: `SKILL.md`
- Reference files: `references/query-generation.md`, `references/search-endpoint.md`
- Difference from qodo-get-rules: Uses `POST /rules/search` instead of `GET /rules` with pagination

**`skills/qodo-pr-resolver/`:**
- Purpose: Fetches Qodo review issues from PR/MR, fixes them interactively or in batch
- Contains: Skill definition, provider-specific command reference
- Entry point: `SKILL.md`
- Reference files: `resources/providers.md` (GitHub, GitLab, Bitbucket, Azure DevOps specific commands)

**`.claude/`:**
- Purpose: Local Claude Code settings and skill installations
- Contains: `settings.local.json` (local overrides), `skills/` (local skill copies during development)
- Generated by: `npx skills add` during local testing
- Not committed: Excluded via `.gitignore`

**`hooks/`:**
- Purpose: Git hooks for automation (pre-commit checks, etc.)
- Contains: Bash scripts for validation and code checks

**`.planning/codebase/`:**
- Purpose: GSD analysis documents (generated by `/gsd:map-codebase`)
- Contains: ARCHITECTURE.md, STRUCTURE.md, CONVENTIONS.md, TESTING.md, CONCERNS.md
- Generated by: GSD orchestrator, consumed by `/gsd:plan-phase` and `/gsd:execute-phase`

## Key File Locations

**Entry Points:**

- `skills/qodo-get-rules/SKILL.md`: Agent-facing instructions for fetching all rules
- `skills/qodo-get-relevant-rules/SKILL.md`: Agent-facing instructions for semantic rule search
- `skills/qodo-pr-resolver/SKILL.md`: Agent-facing instructions for PR review resolution
- `README.md`: User-facing documentation and installation guide
- `AGENTS.md`: Developer-facing guidelines (universal, applies to all skills)
- `CONTRIBUTING.md`: Contribution workflow and skill authoring best practices

**Configuration:**

- `AGENTS.md`: Universal skill guidelines, file size limits, testing requirements
- `CLAUDE.md`: Claude Code-specific tool usage, commit conventions, parallel execution
- `skills/qodo-get-rules/AGENTS.md`: Skill-specific agent guidelines (TBD)
- `skills/qodo-get-relevant-rules/AGENTS.md`: Skill-specific agent guidelines (query generation strategy)
- `skills/qodo-pr-resolver/AGENTS.md`: Skill-specific agent guidelines (TBD)

**Core Logic:**

No source code files (*.py, *.sh, *.cmd) are committed to the repository. Skills use:
- Step-by-step instructions in SKILL.md (executed by agent)
- Reference documents for detailed specifications (pagination, query format, provider commands)
- Python/Bash scripts are executed directly by agents via Bash tool (no scripts directory in repo)

**Documentation & References:**

- `skills/qodo-get-rules/references/repository-scope.md`: How to extract repo scope from git remote
- `skills/qodo-get-rules/references/pagination.md`: Pagination algorithm for GET /rules
- `skills/qodo-get-rules/references/output-format.md`: Expected format for rule output
- `skills/qodo-get-relevant-rules/references/query-generation.md`: How to generate search queries (Name/Category/Content format)
- `skills/qodo-get-relevant-rules/references/search-endpoint.md`: POST /rules/search contract and top_k behavior
- `skills/qodo-pr-resolver/resources/providers.md`: Commands for each git provider (GitHub, GitLab, Bitbucket, Azure DevOps)

## Naming Conventions

**Files:**

- `SKILL.md`: Agent Skills standard format (always uppercase, always at skill root)
- `AGENTS.md`: Universal agent guidelines or skill-specific context (root and skill-level)
- `CLAUDE.md`: Claude Code-specific directives (root and skill-level)
- `references/[name].md`: Detailed specifications, algorithms, or lookup tables
- `resources/[name].md`: Provider-specific commands and configuration

**Directories:**

- `skills/qodo-*`: Skill directories follow `qodo-` prefix convention (matches Agent Skills naming)
- `references/`: Reference documents within skill directories
- `resources/`: Resource/lookup documents within skill directories
- `.planning/codebase/`: GSD analysis documents (generated)
- `.tmp/`: Temporary files, test results (git-ignored)

## Where to Add New Code

**New Skill:**

1. Create `skills/qodo-[skill-name]/` directory
2. Write `skills/qodo-[skill-name]/SKILL.md` with:
   - YAML frontmatter: `name:`, `description:`, `triggers:`, `allowed-tools:`
   - Step-by-step workflow (numbered steps)
   - Error handling and edge cases
   - Configuration section
3. Create `skills/qodo-[skill-name]/AGENTS.md` if skill has unique patterns (reference layer, testing strategy, etc.)
4. Create `skills/qodo-[skill-name]/references/` directory for detailed specs
5. Write reference docs (algorithms, command lookups, output formats)

**New Reference Document:**

- Location: `skills/[skill-name]/references/[topic].md` or `resources/[topic].md`
- Purpose: Detailed specification, algorithm, or lookup table
- Usage: Referenced from SKILL.md step instructions

**New Skill-Specific Guideline:**

- Location: `skills/[skill-name]/AGENTS.md`
- Purpose: Agent developer guidelines specific to this skill (testing, development setup, design decisions)
- Size limit: ~150 lines (keep it small, reference root AGENTS.md for universal guidelines)
- Template: Reference `/Users/eilon-baer/Projects/qodo-skills/skills/qodo-get-relevant-rules/AGENTS.md` (58 lines) as example

**Utilities or Shared Code:**

- Not applicable: This is a skill collection with no shared source code
- Each skill is independent and self-contained
- Common patterns are documented in reference files (not shared code)

- Committed: Yes, for team consistency

## File Size Guidelines

**Critical:** Maximum 500 lines per file, ideal ~300 lines

- `AGENTS.md` (root): ~316 lines → at limit, future skills should use skill-level AGENTS.md
- `SKILL.md` files:
  - `qodo-get-rules`: 122 lines ✓
  - `qodo-get-relevant-rules`: 128 lines ✓
  - `qodo-pr-resolver`: 325 lines (within limit, but near 400)
- `README.md`: ~300 lines ✓
- `resources/providers.md`: 329 lines ✓
- `references/query-generation.md`: 105 lines ✓

**When exceeding limits:**

1. Extract detailed specs to `references/` or `resources/` directories
2. Create skill-specific `AGENTS.md` for domain-specific guidelines
3. Use progressive disclosure: Point to detailed docs instead of inlining everything
4. Break reference documents by topic (one reference per concern)
